## 详细修复方案

### 修复1: 专家会诊思考面板内容错位

**方案**: 修改 `court_agents.py` 的 `speak()` 方法，增加流式过程中的 content 检测和 fallback。

```python
# court_agents.py speak() 方法修改要点：
async def speak(self, context, case, history, temperature=0.4):
    # ... 现有代码 ...
    content_acc = ""
    reasoning_acc = ""
    has_yielded_content = False

    try:
        async for kind, text in self.gateway.chat_stream_with_reasoning(...):
            if kind == "reasoning":
                reasoning_acc += text
                yield ("reasoning", text)
            else:  # content
                content_acc += text
                has_yielded_content = True
                yield ("content", text)
    except LLMGatewayError:
        if not content_acc:
            yield ("content", f"（{self.role_label}暂无回应，LLM 调用失败）")

    # 关键修复：如果流式过程中没有 yield 任何 content，立即用 reasoning 作为 content
    # 而不是等到流式结束后一次性 yield（这会导致流式过程中发言区域为空）
    if not has_yielded_content and reasoning_acc:
        # 将 reasoning 分段 yield，模拟流式效果
        chunk_size = 50
        for i in range(0, len(reasoning_acc), chunk_size):
            yield ("content", reasoning_acc[i:i+chunk_size])
```

**ThinkingPanel.tsx 修改**: 实现智能合并逻辑，将短碎片合并为段落。

```typescript
// ThinkingPanel.tsx 新增 mergeSteps 函数
function mergeSteps(steps: string[]): { type: 'step' | 'paragraph', text: string }[] {
  const result: { type: 'step' | 'paragraph', text: string }[] = []
  let paragraphBuffer = ''

  for (const step of steps) {
    // 完整的编排提示（以句号结尾或长度>20）保留为 step
    if (step.length > 20 || step.endsWith('...') || step.endsWith('。')) {
      if (paragraphBuffer) {
        result.push({ type: 'paragraph', text: paragraphBuffer })
        paragraphBuffer = ''
      }
      result.push({ type: 'step', text: step })
    } else {
      // 短碎片合并为段落
      paragraphBuffer += step
    }
  }
  if (paragraphBuffer) {
    result.push({ type: 'paragraph', text: paragraphBuffer })
  }
  return result
}
```
