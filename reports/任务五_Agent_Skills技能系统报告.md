任务五 Agent Skills 技能系统报告

本报告记录了红岩网校2026考核任务五的完成情况，涵盖Agent技能系统的设计与实现，包括技能的定义格式、发现机制、匹配逻辑、执行方式，以及具体法律领域技能的开发实践。

律社项目的技能系统旨在解决Agent工具管理和领域知识封装的问题。随着Agent功能的扩展，工具数量不断增加，直接将所有工具暴露给Agent会导致选择困难和上下文膨胀。技能系统将相关的工具和知识打包成独立的技能单元，Agent可以根据任务需求动态加载和使用，既提高了效率，也增强了系统的可扩展性。

技能系统的核心设计采用目录加配置文件的方式。每个技能是一个独立的目录，包含一个SKILL.md文件作为技能的描述文件，以及相关的参考资料和辅助文件。SKILL.md采用Markdown格式，开头包含YAML格式的元数据头部，用于存储技能的结构化信息，包括名称、描述、版本、作者、兼容性等。元数据之后是详细的说明文档，包括技能用途、适用场景、工具列表、输出格式、注意事项等内容。这种设计兼顾了人类可读性和机器可解析性，开发者可以直接阅读Markdown文件了解技能详情，系统也可以通过解析YAML元数据获取技能的结构化信息。

技能注册表是技能系统的核心组件，负责技能的发现和注册。系统启动时，注册表会自动扫描技能目录下的所有子目录，查找并解析每个技能的SKILL.md文件，提取元数据和文档内容，创建技能对象并注册到系统中。这种自动发现机制使得新增技能无需修改核心代码，只需在技能目录下添加新的技能包即可，大大降低了功能扩展的成本。注册表还提供技能查询、搜索和列表等接口，支持按名称、关键词、类别等方式检索技能。

技能匹配器负责根据用户的输入和任务场景，判断应该激活哪些技能。匹配逻辑结合了关键词匹配和语义匹配两种方式。关键词匹配基于技能描述中的关键词进行快速筛选，具有速度快、准确率高的优点，但无法处理同义词和语义相关的表达。语义匹配通过向量相似度计算，能够识别语义相关但字面不同的输入，匹配范围更广，但计算成本相对较高。系统采用两级匹配策略，先通过关键词匹配进行快速筛选，再用语义匹配进行精确判断，兼顾了效率和准确性。匹配结果按照相关度排序，选择最相关的技能进行激活。

技能执行器负责技能的加载和执行。当匹配到相关技能后，执行器会加载该技能，提取其中的工具定义和知识内容，注入到当前Agent的上下文中。工具注入将技能包含的工具函数注册到Agent的可用工具列表中，使Agent能够调用这些工具。知识注入将技能包含的领域知识、操作指南、最佳实践等内容添加到Agent的系统提示中，指导Agent更好地使用技能完成任务。注入完成后，Agent就可以像使用原生能力一样使用技能提供的功能，无需区分是内置能力还是技能能力。

项目目前实现了三个核心法律技能，分别是合同风险审查技能、法律检索技能和法律咨询技能。合同风险审查技能专门用于合同文本的风险分析，包含完整的合同审查清单、风险分类标准、输出格式规范和安全提示。该技能提供了结构化的审查框架，涵盖合同主体、标的条款、金额支付、交付验收、违约责任、争议解决、解除终止等多个审查维度，确保审查的全面性和专业性。法律检索技能专注于法律资料的检索，包含法律分类体系、检索策略指南和结果整理规范。该技能能够指导Agent进行更精准的法律检索，包括确定法律类别、提取关键概念、使用多种表述方式、验证时效性等，提高检索的准确率和全面性。法律咨询技能用于一般性的法律咨询服务，包含咨询回答框架、风险提示模板和追问引导策略。该技能规范了法律咨询的回答结构，包括情况总结、法律分析、建议意见、风险提示等部分，确保回答的条理性和完整性，同时要求主动追问不明确的信息，避免基于不完整事实给出建议。

技能系统与Agent的集成为动态加载模式。用户输入问题后，系统首先通过匹配器识别相关技能，然后执行器加载这些技能并注入到Agent中，Agent带着技能提供的工具和知识去完成任务。这种方式使得Agent无需加载所有技能，只需加载与当前任务相关的技能，减少了上下文长度，提高了工具选择的准确率。同时，技能的独立开发和热插拔特性也使得系统的功能扩展更加灵活。

版本迭代过程中，技能系统经历了从硬编码工具到模块化技能的演进。初期所有工具都直接硬编码在Agent中，新增工具需要修改Agent代码，维护成本高。后续引入了工具类的封装，将相关工具组织成类，改善了代码结构。技能概念引入后，系统开始按照能力域组织工具和知识，形成了技能的雏形。随着SKILL.md格式的确定和自动发现机制的实现，技能系统逐步成熟。技能匹配和动态加载功能的加入，使得技能真正成为可插拔的能力单元。最后通过实际技能的开发和调试，不断优化技能系统的设计和实现。

技能系统的引入为律社项目带来了多方面的价值。在代码组织方面，按技能划分的模块结构使得代码更加清晰，每个技能职责明确，便于维护和测试。在能力扩展方面，新增技能无需修改核心代码，只需添加独立的技能包，大大降低了扩展成本。在领域知识管理方面，技能为领域知识提供了标准化的承载方式，使得专业知识能够系统化地沉淀和复用。在系统架构方面，技能系统形成了平台加技能的架构模式，Agent平台提供通用能力，技能提供领域能力，两者分离使得系统更加灵活和可扩展。

通过本次技能系统的设计与实现，深入理解了Agent能力模块化的重要性和实现方法。技能系统不仅是工具的打包机制，更是领域知识的载体和能力扩展的框架。这种平台加技能的架构模式，对于构建复杂的领域AI系统具有重要的参考价值。随着技能生态的不断丰富，Agent的能力边界也将不断扩展，能够处理更多更复杂的法律任务。
